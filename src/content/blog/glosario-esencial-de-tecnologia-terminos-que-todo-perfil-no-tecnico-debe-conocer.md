# Glosario esencial de tecnología

Trabajar en una compañía de tecnología sin un trasfondo técnico puede ser un reto. Las conversaciones suelen estar cargadas de acrónimos, jergas y conceptos que parecen complejos, pero que en realidad pueden entenderse con explicaciones claras. Este glosario está diseñado para que perfiles no técnicos, como gerentes de producto, marketers o profesionales de negocio, puedan comunicarse mejor con equipos de desarrollo, infraestructura, producto y ciberseguridad, fomentando una colaboración efectiva y reduciendo malentendidos.

---

## Conceptos generales

- **Backend**: Parte de una aplicación que corre en el servidor y gestiona la lógica, bases de datos y procesos internos. Es “lo que no se ve” pero hace que todo funcione.
- **Frontend**: Parte de la aplicación que interactúa directamente con el usuario (pantallas, botones, formularios). Es “lo que se ve” y define la experiencia de usuario.
- **Fullstack**: Profesional o enfoque que combina conocimientos de frontend y backend, abarcando todo el ciclo de desarrollo.
- **API (Application Programming Interface)**: Conjunto de reglas que permiten que dos sistemas distintos se comuniquen entre sí. Ejemplo: el login con Google o Facebook en una app.
- **Framework**: Conjunto de herramientas, bibliotecas y reglas predefinidas que agilizan el desarrollo de aplicaciones. Ejemplos: React (frontend), Angular (frontend), Django (backend).
- **Librería**: Colección de funciones o código reutilizable para tareas específicas dentro de un proyecto, como cálculos o manipulaciones de datos.
- **Script**: Archivo de código que ejecuta una serie de instrucciones automáticas, como automatizar tareas repetitivas.
- **Deploy (Despliegue)**: Acción de poner en producción una aplicación o actualización para que los usuarios puedan acceder a ella.
- **Bug**: Error en el software que provoca un mal funcionamiento o resultados inesperados.
- **Patch**: Corrección puntual de un bug o vulnerabilidad en el software.
- **Release**: Versión publicada de un software, que puede incluir nuevas funcionalidades o mejoras.
- **Refactorización**: Proceso de mejorar el código sin cambiar su funcionalidad, para hacerlo más legible, eficiente o mantenible.
- **Versionado**: Sistema para gestionar diferentes versiones de un software o código, como Git o SVN.

## Infraestructura y operaciones

- **Servidor**: Computadora (física o virtual) que ejecuta aplicaciones, almacena datos o sirve contenido a otros dispositivos o usuarios.
- **Nube (Cloud)**: Infraestructura tecnológica que se alquila bajo demanda, proporcionando flexibilidad y escalabilidad. Ejemplos: AWS, Azure, Google Cloud Platform (GCP).
- **Contenedor (Docker)**: Tecnología que empaqueta aplicaciones con sus dependencias para que funcionen de manera consistente en cualquier entorno.
- **Orquestador (Kubernetes)**: Sistema que automatiza la gestión, despliegue y escalado de múltiples contenedores.
- **DevOps**: Cultura y conjunto de prácticas que integran desarrollo (Dev) y operaciones (Ops) para acelerar entregas y mejorar la confiabilidad del software.
- **CI/CD (Integración y Entrega Continua)**: Automatización de procesos para probar, construir y desplegar software de forma rápida y segura.
- **Escalabilidad**: Capacidad de un sistema para manejar un aumento de carga (usuarios, datos, tráfico) sin comprometer su rendimiento.
- **Latencia**: Tiempo que tarda un sistema en responder a una solicitud, medido en milisegundos.
- **Downtime**: Periodo en que un sistema, aplicación o servicio no está disponible para los usuarios.
- **Load Balancer (Balanceador de carga)**: Sistema que distribuye el tráfico entre varios servidores para optimizar el rendimiento y evitar sobrecargas.
- **Monitoreo**: Proceso de supervisar el rendimiento, disponibilidad y salud de sistemas mediante herramientas como Prometheus o Grafana.

## Bases de datos

- **Base de datos relacional (SQL)**: Organiza la información en tablas con filas y columnas, ideal para datos estructurados. Ejemplos: MySQL, PostgreSQL, Oracle.
- **Base de datos NoSQL**: Almacena información de forma flexible (documentos, grafos, clave-valor), ideal para datos no estructurados o semi-estructurados. Ejemplos: MongoDB, Cassandra.
- **Query (Consulta)**: Instrucción escrita en un lenguaje como SQL para obtener, modificar o eliminar datos en una base de datos.
- **Indexación**: Proceso de optimizar el almacenamiento de datos para acelerar las consultas, reduciendo el tiempo de búsqueda.
- **Replica**: Copia de una base de datos que se usa para balancear carga, mejorar disponibilidad o como respaldo en caso de fallos.
- **Backup**: Copia de seguridad de los datos para protegerlos contra pérdidas o fallos del sistema.
- **Transacción**: Conjunto de operaciones en una base de datos que se ejecutan como una unidad, garantizando consistencia (ejemplo: transferencia bancaria).

## Seguridad y buenas prácticas

- **Autenticación**: Proceso para verificar la identidad de un usuario, como ingresar usuario y contraseña o autenticación multifactor (MFA).
- **Autorización**: Proceso que determina qué permisos tiene un usuario autenticado dentro de un sistema (ejemplo: acceso a ciertas funcionalidades).
- **Encriptación**: Técnica para proteger datos convirtiéndolos en un formato ilegible sin una clave específica. Ejemplo: HTTPS usa encriptación.
- **Token**: Código único generado para autenticar y autorizar accesos en sistemas, como en sesiones de usuario o APIs.
- **Phishing**: Ataque cibernético que engaña a usuarios para robar credenciales o información sensible, como correos falsos que simulan ser de una fuente confiable.
- **Firewall**: Sistema que filtra tráfico de red para proteger servidores o aplicaciones de accesos no autorizados.
- **Vulnerabilidad**: Debilidad en un sistema que puede ser explotada por atacantes para comprometer su seguridad.
- **Parche de seguridad**: Actualización diseñada para corregir vulnerabilidades específicas en un sistema o software.
- **OWASP**: Proyecto que identifica las principales amenazas de seguridad en aplicaciones web y propone buenas prácticas para mitigarlas.

---

## Conclusión

Entender estos términos no significa volverse programador, pero sí permite conversar con mayor claridad con equipos técnicos, tomar decisiones mejor informadas y evitar malentendidos. Un lenguaje común es un puente entre negocio y tecnología, y este glosario es el primer paso para construirlo. Al dominar estos conceptos, los profesionales no técnicos pueden participar activamente en discusiones técnicas, alinear objetivos y contribuir al éxito de proyectos tecnológicos.
